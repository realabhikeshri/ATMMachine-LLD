## Overview
This project implements a production-grade ATM Machine using
clean object-oriented principles and State Design Pattern.

## Features
- Card insertion & PIN authentication
- Balance inquiry
- Cash withdrawal with denominations
- Account & ATM cash validation
- Safe rollback on failure

## Design Patterns
- State Pattern (ATM workflow)
- Single Responsibility Principle
- Open Closed Principle

## Key  Highlights
- ATM cash vs Account balance separation
- Safe rollback when ATM cash is insufficient
- Extendable for:
  - Deposit
  - Mini statement
  - Multiple accounts
  - Bank service integration

## How to Run
1. Open solution in Visual Studio
2. Run Program.cs
3. Modify scenarios as needed
